## General Questions

### What is AgentVectorDB?
AgentVectorDB is a specialized vector database designed for AI agents. It provides:
- Efficient memory storage and retrieval
- Semantic search capabilities
- Async/sync API support
- Built-in importance scoring
- Metadata management

### Why use AgentVectorDB?
- **Purpose-built for agents**: Optimized for agent memory management
- **Lightweight**: Minimal dependencies and resource usage
- **Flexible**: Works with any embedding model
- **Fast**: Built on LanceDB for high-performance vector operations
- **Easy to use**: Simple, intuitive API

### What's the maximum vector dimension supported?

There's no hard limit, but we recommend:
- 64-512 dimensions for simple use cases
- 768-1024 dimensions for transformer models
- Up to 4096 dimensions for specialized cases

> [!note]
> Higher dimensions increase memory usage and processing time.

### How do I handle concurrent access?

Use the async API for concurrent operations:

```python
from agentvectordb import AsyncAgentVectorDBStore

async def safe_access():
    store = AsyncAgentVectorDBStore()
    collection = await store.get_or_create_collection("shared_memories")
    await collection.add(content="Concurrent access test")
```

## Performance Questions

### What's the recommended batch size for additions?

We recommend:
- 100-1000 items per batch for normal usage
- 50-100 items for real-time applications
- Up to 5000 items for bulk imports

```python
def optimal_batch_add(collection, items, batch_size=1000):
    for i in range(0, len(items), batch_size):
        batch = items[i:i + batch_size]
        collection.add_batch(batch)
```

### How can I optimize query performance?

1. Use efficient filters:
```python
# Good
results = collection.query(
    query_text="important",
    filter_sql="importance_score > 0.8"
)

# Better (using indexed fields)
results = collection.query(
    query_text="important",
    filter_sql="type = 'critical' AND metadata.priority = 'high'"
)
```

## Error Handling

### How do I debug common errors?

1. Connection Issues:
```python
try:
    store = AgentVectorDBStore(db_path="./my_db")
except InitializationError as e:
    print(f"Database initialization failed: {e}")
```

2. Query Errors:
```python
try:
    results = collection.query(query_text="test")
except QueryError as e:
    print(f"Query failed: {e}")
```

### How do I handle migration errors?

Use our migration utility:
```python
from agentvectordb.utils import migrate_database

try:
    migrate_database("old_db_path", "new_db_path")
except OperationError as e:
    print(f"Migration failed: {e}")
```
